# מדריך בדיקה — בורר חשבונות חיוב (PR #90 ו-#92)

מדריך לבדיקה ידנית בדפדפן. עבור צעד-צעד וסמן ✅/❌ ליד כל שורה.

> **דרישה מקדימה (חשוב):** צריך חברה עסקית (B2B) שיש לה **לפחות 2 חשבונות חיוב**,
> ולפחות הזמנה/תעודת משלוח אחת המשויכת לכל חשבון. בלי נתונים כאלה אין מה לראות במסך.
> אם אין כזו חברה בחנות-הבדיקה — צריך קודם להקים אחת דרך מסך הניהול → "ארגונים".

---

## חלק א' — הקופה (PR #90)

נכנסים לחנות כלקוח של חברה עם כמה חשבונות, מוסיפים מוצר לעגלה, וממשיכים לקופה.

באזור **"01 — פרטי לקוח"**, בשדה **"מספר לקוח"**:

- [ ] הרשימה הנפתחת **פעילה** (אפשר ללחוץ ולפתוח אותה) — בעבר היא הייתה תקועה/אפורה.
- [ ] מופיעים בה **כל החשבונות** של החברה, כל אחד בתבנית `מספר — שם`.
- [ ] **החשבון הראשי מסומן אוטומטית** כברירת מחדל.
- [ ] מתחת לרשימה מופיע מונה נכון, למשל "3 חשבונות רשומים בחברה שלכם".
- [ ] אפשר **לבחור חשבון אחר** מהרשימה.
- [ ] משלימים הזמנה, ואז נכנסים אליה במסך הניהול → ההזמנה מציגה את **החשבון שנבחר** (ולא תמיד הראשי).

**בדיקת קצה:** לקוח של חברה עם **חשבון אחד בלבד** —
- [ ] השדה מופיע אך **נעול לקריאה בלבד** (אי אפשר לשנות), והכיתוב "חשבון אחד רשום בחברה שלכם".

**בדיקת קצה:** לקוח רגיל (לא עסקי, בלי חברה) —
- [ ] שדה "מספר לקוח" **לא מופיע בכלל**. הקופה עובדת כרגיל.

---

## חלק ב' — כרטסת הלקוח בניהול (PR #92)

מסך ניהול → ארגונים → בוחרים חברה עם כמה חשבונות → פותחים את **כרטסת הלקוח**.

בראש החלון:

- [ ] מופיע **בורר חשבונות**: כפתור "כל החשבונות" + כפתור לכל חשבון (שם + מספר).
- [ ] כברירת מחדל מסומן **"כל החשבונות"**, ורואים את כל המסמכים.
- [ ] לוחצים על חשבון מסוים → **רשימת המסמכים מסתננת** ומציגה רק את המסמכים של אותו חשבון.
- [ ] **הסכומים** (סך הכל / יתרה) מתעדכנים לפי החשבון שנבחר.
- [ ] **מונה הלשונית** מתעדכן בהתאם.
- [ ] שדה **"מספר לקוח"** בכותרת מציג את מספר החשבון שנבחר (או "הכל").
- [ ] בחירת מסמכים ל**הפקת חשבונית מרוכזת** מוגבלת לחשבון שנבחר.

**בדיקת קצה:** חברה עם **חשבון אחד בלבד** —
- [ ] בורר החשבונות **לא מופיע** (אין מה לסנן), והכרטסת עובדת כרגיל.

## אם משהו נכשל
סמן ❌ ליד השורה, צלם מסך, ושלח לי — אבדוק ואתקן (אם זה תיקון בסמכותי) או אעביר לפיליפ.
